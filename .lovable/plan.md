

# Единый контейнер для контента

## Что делаем

Сейчас каждая секция задаёт свою ширину и отступы (`px-[70px]`, `max-w-[1425px]`, фиксированные `w-[625px]` и т.д.) независимо друг от друга --- из-за этого контент "гуляет" по ширине. Нужно привести всё к единому контейнеру, ширина которого определяется фотогалереей (625 + 23 + 624 = ~1272px контента).

## Подход

1. **Добавить утилитарный CSS-класс `.content-container`** в `src/index.css`:
   - `max-width: 1280px`, `margin: 0 auto`
   - Адаптивные отступы: `px-4` на мобилке, `px-8` на планшете, `px-[70px]` на десктопе

2. **Обновить `Index.tsx`** --- обернуть весь `<main>` в контейнер, убрать индивидуальные `px-[70px]` из каждого компонента.

3. **Секции с фоном на всю ширину** (FinancialSection, ContactForm, Footer) --- фон остаётся full-width, но внутренний контент оборачивается в тот же контейнер. Для этого из секции убираем `px-[70px]`, а внутрь добавляем `div` с классом контейнера.

4. **Убрать фиксированные ширины** из дочерних элементов, где возможно, заменив на `flex-1`, `w-full`, проценты --- чтобы контент растягивался/сжимался внутри контейнера.

## Адаптивность

Контейнер будет использовать Tailwind breakpoints:
- **Mobile** (< 768px): `px-4` (16px)
- **Tablet** (768-1024px): `px-8` (32px)
- **Desktop** (> 1024px): `px-[70px]`
- `max-width: 1280px` + `mx-auto` на всех размерах

Это подготовит основу для дальнейшей мобильной адаптации отдельных компонентов.
