

## Итоги аудита: кнопки на страницах объектов

На страницах объектов (`/objects/:slug`) есть следующие кнопки/ссылки, которые нужно проверить:

### Уже работают корректно
| Элемент | Поведение |
|---------|-----------|
| Презентация (PropertyDetails) | Настраивается через `presentation_url` в админке |
| Купить паи (PropertyDetails) | Открывает форму заявки — работает |
| Графики: «Купить паи» (FinancialSection) | Настраивается через `buy_url` в админке |
| Графики: «Связаться с нами» (FinancialSection) | Настраивается через `contact_url` в админке |

### Требуют исправления на страницах объектов

| Элемент | Проблема | Решение |
|---------|----------|---------|
| **«Поделиться»** (PropertyHero) | `<button>` без действия | Добавить копирование текущего URL в буфер обмена (с toast «Ссылка скопирована») |
| **Карточки «Похожие объекты»** (SimilarProperties) | Захардкожены, `<div>` без ссылки, не ведут никуда | Заменить на динамические данные из БД (другие опубликованные страницы) и обернуть в `<a href="/objects/{slug}">` |
| **«Подробнее»** в popup карты (InvestmentMap) | `href="#"` — заглушка | Пока не на странице объекта, но видна на главной. Можно оставить на потом |

### Также на главной и в футере (не на страницах объектов, но тоже битые)

| Элемент | Проблема | Решение |
|---------|----------|---------|
| «Смотреть все →» (FeaturedProperty) | `href="#"` | Заменить на `/catalog` |
| Карточки новостей (NewsSection) | `href="#"` | Добавить поле `link` в админке |
| Карточки блога (BlogSection) | `href="#"` | Добавить поле `link` в админке |
| Карточки Telegram (TelegramSection) | `href="#"` | Добавить поле `link` в админке |
| Ссылки Footer (Категории, О компании, Политики) | `href="#"` | Проставить реальные ссылки |
| «Личный кабинет» (Header) | Кнопка без действия | Заменить на «Оставить заявку» с формой (из предыдущего плана) |

## План реализации (приоритет — страницы объектов)

### 1. PropertyHero: кнопка «Поделиться»
- Добавить `onClick` → `navigator.clipboard.writeText(window.location.href)` + toast «Ссылка скопирована»

### 2. SimilarProperties: динамические данные и ссылки
- Загружать 3 случайных опубликованных объекта из БД (исключая текущий slug)
- Для каждого брать данные из блоков `PropertyHero` (title), `PhotoGallery` (первое фото), `PropertyStats` (характеристики)
- Обернуть каждую карточку в `<a href="/objects/{slug}">`
- Если объектов меньше 3 — показывать сколько есть; если 0 — скрывать секцию

### 3. FeaturedProperty: «Смотреть все →»
- Заменить `href="#"` на `href="/catalog"`

### 4. Header: «Личный кабинет» → «Оставить заявку»
- Как в ранее одобренном плане: заменить на кнопку с модальной формой

### 5. Footer: проставить реальные ссылки
- «Все объекты» → `/catalog`
- «Пайстартер сегодня» / «О команде» → `/about`
- Убрать неактуальные категории (Торговые центры, Отели и т.д.) или оставить только «Все объекты»
- Политики: оставить `#` пока нет страниц

### 6. Новости / Блог / Telegram карточки: поле `link` в админке
- В `AdminHomePage` добавить поле «Ссылка» к каждому элементу в секциях Новости, Блог, Telegram
- В компонентах использовать `item.link || "#"` и при пустой ссылке не делать `<a>`, а оставить `<div>`

### Технические детали

**SimilarProperties** — самое существенное изменение:
- Компонент получает `currentSlug` как prop
- `useEffect` загружает из `pages` + `page_blocks` данные для карточек
- SQL: выбрать pages где `status='published'` и `slug != currentSlug`, limit 3, random order
- Для каждой страницы подтянуть блоки PropertyHero, PhotoGallery, PropertyStats через join

**Остальные изменения** — точечные правки в существующих компонентах без изменений в БД.
